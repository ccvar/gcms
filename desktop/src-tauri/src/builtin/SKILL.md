# 网页设计规范

> 视觉质量的下限靠**纪律**保证，不靠临场发挥。这份规范就是那套纪律。
> **不确定怎么好看时的万能解：减色、减框、加留白。**

## 0. 先看有没有起点

动手前先 `ls` 当前目录。**有 `index.html` 就先读它再改，别推倒重写**——它多半是精心设计过的起始模板，`:root` 里已经有整套设计变量。你的活是**替换内容、按需增删区块、微调色板**，不是重做一遍。

## 1. 设计变量契约（硬性）

所有颜色/圆角**只准**从 `:root` 取，正文里不许出现硬编码 hex。变量名固定这几个：

```css
:root {
  --bg: #ffffff;        /* 页面底 */
  --surface: #f7f7f5;   /* 卡片/区块底，与 --bg 拉开一点点 */
  --text: #1a1a1a;      /* 正文 */
  --muted: #6b6b6b;     /* 次要文字，仍需 ≥4.5:1 */
  --accent: #2f6fed;    /* 唯一强调色 */
  --radius: 8px;
  /* 间距刻度：只用这些值，不许 13px/17px 这种随手数 */
  --s1: 4px; --s2: 8px; --s3: 12px; --s4: 16px; --s5: 24px; --s6: 40px; --s7: 64px;
}
```

**改色板 = 改这几个变量**，不是全站搜替换。

## 2. 颜色

- **中性色 + 恰好一个强调色**。用户没点名要多彩就别自作主张加第二个强调色。
- 强调色只用在：主 CTA、链接、极少量点缀。**不要**每个标题都染色、不要每个图标都染色。
- **正文对比度 ≥ 4.5:1**（WCAG AA）。深色底尤其容易翻车——浅灰字放近黑底上经常只有 3:1。拿不准就把 `--muted` 调深。
- 大标题（≥24px 且加粗）可放宽到 3:1，但别拿这个当偷懒借口。

## 3. 排版

- **字号成体系**：14 / 16 / 20 / 28 / 40。别只有"大标题 + 正文"两档——层级塌了页面就显廉价。
- **正文行高 1.6–1.8**；标题 1.1–1.3。
- **字体成对选**：同族不同字重，或经典搭配（无衬线标题+无衬线正文、衬线标题+无衬线正文）。**别引一堆字体**。
- **中文站用系统字体栈**（离线可用、不闪字）：
  ```css
  /* 无衬线 */ font-family: system-ui, -apple-system, "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif;
  /* 衬线   */ font-family: Georgia, "Songti SC", "SimSun", "Noto Serif SC", serif;
  /* 等宽   */ font-family: ui-monospace, SFMono-Regular, Menlo, Consolas, monospace;
  ```
- 正文**每行 45–75 字符**最舒服 → 长文窄栏约 680–760px。

## 4. 布局与节奏

- 内容有**最大宽度容器**（常用 680–1100px）居中。长文取窄端，落地页取宽端。
- **区块间留白宁大勿小**：区块之间 `--s7`(64px) 起步，区块内元素之间 `--s4`/`--s5`。
- **节奏要匀**：同级元素间距必须一致。页面显得"脏"，八成是间距没对齐到刻度。
- 别每个区块都加边框+阴影+底色。**层次靠留白和字重做**，框和影是最后手段。

## 5. 组件与状态

- **所有可点元素都要有 `:hover`**，还要有 `:focus-visible`（键盘可达性，别用 `outline:none` 一删了事）。
- 按钮：主/次/文字链三级足矣。主 CTA 全页**只有一个**最显眼的。
- 表单：label 永远在（placeholder 不是 label）；错误态给文字，不只是变红。
- 别忘了**空态/加载态/错误态**——只做"一切正常"的页面是半成品。
- 图标用**内联 SVG**，别用 emoji 充数。

## 6. 响应式

- 至少测 **1280**（桌面）和 **390**（手机）两个宽度。
- 手机端：不许横向溢出；正文别小于 15px；点击区 ≥44px；多列塌成单列。
- 断点按内容需要来（常用 640/768/1024），不是背设备尺寸。

## 7. 成品自检（硬性，交付前必做）

用随附的截图工具截 **1280** 和 **390×844** 两张，**用 Read 打开图逐项看**：

- [ ] 横向溢出/错位
- [ ] 文字对比度够不够（尤其 `--muted` 和深色底）
- [ ] 间距节奏是否匀、有没有跑刻度的值
- [ ] 字体真的生效了吗（别是回退到默认宋体还不自知）
- [ ] 图片/SVG 有没有裂
- [ ] hover / focus 态在不在
- [ ] 手机端有没有挤爆、字有没有过小

**发现问题先修再截一轮确认。别把没亲眼看过的页面交给用户。**

## 8. AI 做站最常翻车的点（对号入座）

1. **每个区块都套卡片**——加边框、加阴影、加底色。结果整页全是盒子，没有呼吸。→ 大部分区块就该是**裸的**。
2. **强调色滥用**——标题、图标、边框、背景全染上。→ 强调色是**稀缺资源**，一页用三五处足够。
3. **字号只有两档**——40px 大标题 + 16px 正文，中间全空。→ 缺 20/28 档，层级就是塌的。
4. **间距随手写**——13px、17px、22px。→ 只准用刻度值。
5. **hero 很猛，下面很干**——头图做得很用力，往下全是干瘪列表。→ 节奏要贯穿全页。
6. **渐变/毛玻璃/动效堆料**——用来掩盖没有排版功底。→ 先把留白和字阶做对。
7. **emoji 当图标**——🚀✨🔥。→ 用内联 SVG。
8. **深色主题只是把底色刷黑**——文字对比度、阴影、边框全没跟着调。→ 深色不是反色。
9. **忘了 `<title>` 和 favicon**——标签页上一片空白。
10. **只做了桌面**——390 一看就散架。

## 9. 参考图

用户贴参考截图时：提取它的**布局结构、配色气质、字重对比、密度**，**不要**抄它的文字、logo、品牌名。做完对照参考图自查气质是否接近。
