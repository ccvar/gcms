Automation API

自动化接口开放文档

这些接口只开放文章、链接、页面。站点设置、分类、导航、安全、系统更新不会通过这里开放。

连接地址

接口根地址：

{{.Site.Root "/api/admin/v1"}}

OpenAPI 描述文件：

{{.Site.Root "/api/admin/v1/openapi.json"}}

认证方式

每个外部工具应单独创建一条访问权限。创建成功后复制访问密钥，请求时任选一种方式传入：

Authorization: Bearer gcms_xxx
X-GCMS-API-Key: gcms_xxx

访问密钥只显示一次。如果泄露或不用了，请在后台吊销对应那一条。

资源列表 / 创建读取 / 更新权限前缀

链接GET/POST /linksGET/PATCH /links/{id}links

权限说明

小权限能做什么例子
读取列出和查看内容posts:read
写草稿创建草稿，修改未发布内容posts:write
发布发布、定时发布、修改已发布内容posts:publish

列表查询参数

参数说明示例
lang内容语种，留空使用默认语种zh
status按状态筛选draft / published / scheduled
q按标题、摘要、正文等关键词查找SQLite
slug按 slug 精确查找about
limit返回数量20
offset分页偏移0

内容字段

字段说明
title标题。创建内容时必填。
content正文 Markdown。
slugURL 里的短路径。留空时会按标题生成。
excerpt摘要。
meta_descSEO 描述。
keywords关键词。
statusdraft、published 或 scheduled。
published_at发布时间。定时发布时必填，使用 RFC3339 或 2006-01-02T15:04。
category_id文章和链接可用；页面不支持。
link_url链接资源的目标地址。

常见用法

修改某篇内容前，先查到准确 ID，不要只凭标题猜。

curl '{{.Site.Root "/api/admin/v1/posts"}}?lang=zh&q=SQLite' \
  -H 'Authorization: Bearer gcms_xxx'

创建一篇文章草稿：

curl -X POST '{{.Site.Root "/api/admin/v1/posts"}}' \
  -H 'Authorization: Bearer gcms_xxx' \
  -H 'Content-Type: application/json' \
  -d '{"title":"AI 生成的草稿","content":"正文 Markdown","lang":"zh","status":"draft"}'

按 ID 更新草稿：

curl -X PATCH '{{.Site.Root "/api/admin/v1/posts/123"}}' \
  -H 'Authorization: Bearer gcms_xxx' \
  -H 'Content-Type: application/json' \
  -d '{"title":"新的标题","meta_desc":"新的 SEO 描述"}'
